# 카페 근무표 자동생성기

직원 4명, 오픈/마감 로테이션 근무표를 자동으로 만들고, 검증하고, 수정하고, 저장/내보내기까지
할 수 있는 로컬 전용 프로그램입니다. 인터넷 연결이 필요 없고, 외부로 어떤 데이터도 전송하지
않습니다.

## 1. 가장 빠르게 써보기 (설치 없이)

`src/index.html` 파일을 더블클릭해서 여세요. 크롬이나 엣지 같은 브라우저에서 바로 열리고,
EXE 버전과 기능이 완전히 동일합니다(저장 위치만 브라우저 내부로 다를 뿐). 먼저 이걸로
테스트해보시고, 마음에 들면 아래 2번으로 진짜 .exe를 만드시면 됩니다.

## 2. 윈도우용 .exe 만들기 (개발자용, 인터넷 필요)

이 폴더를 인터넷이 되는 PC(해미님 개발 PC)로 옮긴 뒤, 터미널에서:

```bash
npm install
npm run dist
```

`dist/` 폴더 안에 `카페근무표-1.0.0-portable.exe` 같은 이름의 단일 실행 파일이 생깁니다.
설치 과정 없이 이 파일 하나만 사장님께 전달하시면, 더블클릭만으로 실행됩니다(USB나
카카오톡 파일전송으로 보내셔도 됩니다).

> 처음 빌드할 때만 인터넷이 필요합니다(Electron 바이너리 다운로드). 빌드가 끝나면
> 만들어진 .exe 자체는 완전히 오프라인으로 동작합니다.

## 3. 사용 방법 (사장님 기준)

1. 상단에서 직원 이름 4명을 입력합니다 (비워두면 직원1~4로 표시됩니다).
2. 만들고 싶은 월을 고르고 **"이 달 일정 생성"** 을 누릅니다. 1초 안에 일정표와 검산표가
   같이 나타납니다.
3. 검산표 맨 위에 ✅ 또는 ⚠️로 전체 통과 여부가 표시되고, 표 안의 빨간 칸이 어떤 조건이
   안 맞는지 보여줍니다.
4. 일정표의 칸을 클릭하면 오픈 → 마감 → 휴무 순서로 바뀝니다. 직접 일부를 바꾸면 검산표가
   즉시 다시 계산되어, 그 수정이 규칙을 깨는지 바로 보여줍니다.
5. 수정 내용은 0.3초 뒤 자동으로 저장됩니다(따로 저장 버튼을 누르지 않아도 됩니다). 프로그램을
   껐다 다시 켜도 마지막으로 보던 달이 그대로 열립니다.
6. 상단 "불러오기" 드롭다운에서 예전에 만든 달을 다시 불러올 수 있습니다.
7. **"이미지로 내보내기"** 는 PNG로, **"표(CSV)로 내보내기"** 는 엑셀에서 바로 열리는
   CSV 파일로 저장합니다.

저장되는 데이터 위치:
- EXE 버전: `%APPDATA%\카페근무표\schedule-data.json` (윈도우 사용자별 앱 데이터 폴더)
- 브라우저 버전: 그 브라우저의 localStorage 안 (다른 브라우저로 열면 안 보입니다)

## 4. 폴더 구조

```
cafe-scheduler/
  package.json          # Electron 빌드 설정
  electron/
    main.js              # 창 생성, 파일 저장/내보내기 (Node fs 사용)
    preload.js            # 렌더러(화면)에 안전하게 기능 노출
  src/
    index.html            # 화면 레이아웃
    style.css
    engine.js             # ★ 규칙/로테이션 계산 엔진 (핵심 로직, 의존성 없음)
    storage.js             # 저장 추상화 (Electron 파일 vs 브라우저 localStorage)
    export.js               # PNG/CSV 내보내기
    app.js                 # 화면 렌더링 + 이벤트 처리
```

## 5. 어떻게 동작하나 (확장하실 때 참고)

`engine.js` 한 파일이 모든 스케줄링 로직을 담당하고, 나머지(UI, 저장, 내보내기)는 전부
이 파일이 돌려주는 결과만 가지고 동작합니다. 그래서 규칙을 바꾸거나 추가하고 싶을 땐
`engine.js`만 보시면 됩니다.

- **`generateSchedule(year, month, cfg, seed)`**: 해당 월에 필요한 전체 주 범위를 계산하고,
  로컬서치(시뮬레이티드 어닐링과 비슷한 방식)로 모든 하드 조건을 만족하는 조합을 찾습니다.
  4명·28일 규모라 보통 1초 이내에 끝납니다.
- **`validateSchedule(days, cfg)`**: 현재 일정표(자동생성이든 사람이 직접 수정한 것이든)를
  받아서 일별/사이클별로 조건 충족 여부를 다시 계산합니다. 셀을 클릭할 때마다 이 함수만
  다시 부르면 되므로 실시간 검산이 가능합니다.
- **`defaultRuleConfig()`**: 오픈/휴무 횟수, 최대 연속근무일수, 주말 오픈/휴무 횟수 등이
  여기 숫자로 모여 있습니다. 조건이 바뀌면 이 객체만 수정하면 됩니다.

### 알아두실 점 — 직원 수를 바꾸고 싶다면

지금 로직은 "직원 4명 / 4주 사이클"에 수학적으로 맞춰져 있습니다(하루 오픈 1명 기준,
한 주 총 오픈 횟수가 7이 되려면 직원 한 명만 그 주에 1회, 나머지 세 명은 2회를 가져야
하는데, 이 조합이 자연스럽게 맞아떨어지는 인원수가 4명입니다). 직원 수를 5명, 6명 등으로
바꾸시려면 `engine.js`의 `targetOpenForWeek` / `targetOffForWeek` 함수의 식을 새로
설계하셔야 합니다. 이 부분이 전체 로직 중 유일하게 인원수에 종속된 부분이라, 코드 안에도
주석으로 표시해두었습니다.

### 자주 나올 만한 확장 아이디어

- 직원별 고정 휴무 요일, 개인 휴가 등 "이 날짜는 무조건 이 값" 같은 고정 셀 기능 →
  `generateSchedule` 호출 전에 `cfg`에 lock 목록을 추가하고, 로컬서치 스왑 시 잠긴 칸은
  건드리지 않도록 `buildWeekPairs`/스왑 로직에 조건만 추가하면 됩니다.
- 진짜 .xlsx 내보내기 → 지금은 CSV(엑셀에서 바로 열림)로 구현되어 있는데, 정식 xlsx
  포맷이 필요하시면 SheetJS 같은 라이브러리를 `npm install xlsx` 후 `export.js`에 추가하시면
  됩니다.
- 직원 수 자체를 UI에서 선택 → `index.html`에 입력칸을 늘리고 `engine.js`의 사이클 수학을
  재설계하면 됩니다(위 항목 참고).

## 6. 한계 / 알려드릴 점

- 이번 버전은 "소프트 선호조건"(오픈한 다음날 휴무, 토·일 같은 사람 오픈)을 100% 보장하지
  않습니다. 하드 조건을 우선 만족시키고, 그 안에서 가능한 만큼 선호도를 반영합니다. 검산표에
  소프트 조건 충족률은 따로 표시하지 않으니, 필요하시면 추가해드릴 수 있습니다.
- 데이터는 PC 한 대(EXE 기준) 또는 브라우저 한 개(HTML 단독 실행 기준)에만 저장됩니다.
  다른 PC와 동기화되지 않습니다.
